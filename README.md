## Задача

Разработано консольное приложение для автоматизации учета товаров на складе. Программа позволяет добавлять, редактировать, удалять, искать, фильтровать и сортировать товары, выводить аналитические показатели, а также сохранять и загружать данные из TXT-файла.

## Технические требования

- Язык программирования: C++17.
- Хранение данных: `std::vector`.
- Формат файла: TXT с разделителем `,`.
- Файл данных: `data/products.txt`.
- Формат строки: `id,name,category,price,quantity`.
- Обработка ошибок ввода: пользовательский ввод проверяется в цикле без аварийного завершения программы.
- Для корректного отображения кириллицы в Windows используется `SetConsoleCP(65001)` и `SetConsoleOutputCP(65001)`.

## Возможности программы

- Добавление товара с уникальным ID.
- Редактирование существующей записи.
- Удаление по ID или названию.
- Валидация цены, количества, строковых полей и уникальности ID.
- Поиск по названию с частичным совпадением.
- Фильтрация по категории и диапазону цен.
- Сортировка по цене, количеству и названию.
- Расчет общей стоимости товаров на складе.
- Вывод товаров с остатком не более 5 штук.
- Вывод топ-3 самых дорогих позиций.
- Автоматическая загрузка данных при старте.
- Ручное сохранение изменений через меню.

## Установка и запуск

### Вариант 1: CMake

```bash
cmake -S . -B build
cmake --build build
./build/WarehouseApp
```

В Windows исполняемый файл может находиться в папке `build/Debug` или `build/Release`, если используется Visual Studio generator.

### Вариант 2: g++

```bash
g++ -std=c++17 src/main.cpp src/Product.cpp src/Warehouse.cpp src/FileIO.cpp src/Menu.cpp -o WarehouseApp
./WarehouseApp
```

## Комментарии по проекту

Самой важной частью проекта была организация логики по отдельным классам: `Product` отвечает за данные товара, `Warehouse` хранит коллекцию и выполняет операции над ней, `FileIO` работает с файлом, а `Menu` отвечает за взаимодействие с пользователем. Такой подход упрощает поддержку и расширение программы.
